Customer details

Customer:

Clarity AS

The purpose of this website is to promote the Inital Coin Offering / Release of Clarity's new crypto valuta. It is a way for interested people to read about Clarity and the event itself, buy tickets for the event, and to contact the organizers.

The goal for the website is to inform potential investors about the new release, enable ticket purchasing and for people to be able to contact Clarity via a contact form

The audience is crypto bros, potential investors and in general people who are interested in crypto and finance.

Navigation Structure

The navigational aspect for the website is based on a Homepage with access to all the subpages using a navigation bar at top and a menu in the websites footer. The navigation bar will be sticky when scrolling down the page, and will therefore be available for the user at any given point of the vertical scroll-position of the website.

The menu will refer the users to the respective subpage with the href-attribute. This will open the subpage in the same window. External link or URLs will take the user to a new window for user-experience motivations. This will allow the user to visit external websites without having to navigate to the same page in out website when done reading at the external website.

We believe this is the best way to structure our navigation because of the amount of subpages. Having the navigation at the top of the screen and in the footer prevents the navbar in distracting the user from the content of the page, while being available at any given time.

The Design Manual

Our color palette is based on the color of the logo, and uses a Chestnut Red as a secondary color, while the primary color is Chinese Black. We use a Yellow Orange color as an accent to give some variation in the color tones shown in the website. This makes a common color-combination of black, yellow and red, which is recogniseable from for instance the German flag. The reasoning behind the color palette is to give a modern look, with great contrast in regards to universal design. It also provides room for sudden accents that will give a neutral, but pleasing color-scheme to the website.

The Content

This chapter will be a text-description of the pages shown above in "The Website Outlay" It will consist of a written description of the elements, design guidelines and methods used to acheive the desired design.

Comments : Design

The website uses a dark background, and strives to a minimalistic design with a red secondary color and uses yellow accents to brighten the site up a little. One of the great advantages of using a minimalistic design is that the users have less elements to control and manage when scanning the site. This will make it easier to find information, while also being aesthetically pleasing fot the eye. We may face some problems regarding visibility, as space is a limited resource, and information needs to be sectioned and categorized in such a way that it is easy and intuitive for the users to find the desired information. Be using a black main color with a red and yellow elements and white text, it makes a quite big contrast, which will make it easier to read. Both the minimalistic design and the color palettes is contributing to fulfilling the universal design for accessibility regarding users with handicaps.

Home

The homepage will be the index, where the users starts at when laoding the website. This will be the "hub" that familiarizes the user to the inition design guidelines and concepts of the website. It is meant to contain all the core content about the event itself, and it should be easy for the users to navigate to the other subpages from the homepage.

It constists of a navigation bar at the top, that will stay sticky for the entite vertical scroll position. This is the same for all the pages, and it acts like a common navigation central for the user, at it is consistent for all the pages. It also has a footer, which will also be in alle the other pages, and consists of a menu to the other pages, and some other common footer-content such as copyright, logo and slogan.

For the relevant content of the page, it is split up in 3 different sections. These are: Header, split-section 1, and split-section 2. The header will consist of a background image and some informative text about the event. It is minimalistic, which prevents the users from getting overwhelmed when entering the site. Being designed in the way it is, it will also give of a serious and elegent impression to the end user. The other two sections are a 50/50 split section, where 50% of the viewport-width is going to be a image, and the other half text. To give a more modern and non-repetetive look, the picture are alternated, such as the left-right in the firstoutlay are mirriored in the second.

About us

The About Us-page will also have a header, that will function as the header on the main page. This content is more directed to the company itself, where the users can get familiar with the company and their story. The sites sections are: header, split-section, text section 1 and text section 2. From header, to split sections, to pure text-paragraph is a nice way to ease to user into more heavy reading. Because space is a limited resource in a minimalistic design, we wanted to give the users as much information as possible, on a small area as possible. This is because we figured that if the users already wants to learn more, they have no constraints on their willingness to read pure text, as in contrast to fancy, space-consuming elements. To get a more modern design, we used assymmetrical headings, with an accent underline, that gives off a sudden, but nice vibe to the headings and change of subject.

Ticket Shop

The Ticket shop is a simple shop, which consists of a sub-menu of the different tickets, and the ticket information itself. We wanted to keep the design in line with the design manual, so we went for a minimalistic design which is kind of the outlay of a paper ticket, to show the ticket type, price, infomation, and price. A CTA button is added on the ticket to indicate for the user that by clicking the button, it will lead to the checkout area. To prevent false affordance, we chose to remove the other buttons, as we dont want to frustrate the users by clicking the other CTA's that just leads to the same site, when the users actually wants to checkout.

Contact Page

The contact page is a non-scroll page, which consists of two elements: google maps and a contact form. Having the form visible to the users on load makes it intuitive for the users of how to use the function. The google maps is an API that uses google maps to pinpoint the location of the company. This is just a common practise when building a contact page, and it is a page not out of the ordinary.

Minimum Requirements

On-scroll Animations

The on-scroll animation will gradually show the content to the users when they scroll through the website. This will be done using JavaScript, by adding an eventlistner and change CSS-classes to determine the styling of the element using JS.

Form-validation and submission

The form-validation and submission functions will enable validation of the input data from the user. This will enable us to secure that the data is in the right format when sending it to the backend-server. The submission-function will be coded, but not implemented, as the project is frontend only, and there wont be a backend server implemented.

Countdown-timer and ticket-counter

The countdown timer will show the remaining time before the event starts, and we will set the counter to count towards the project deadline. This will be done by using dynamic data fields that are sent by JavaScript functions by getting the current time and displaying that on the website.

The ticket-counter will count the remaining tickets for each category. This will be reset each time the user enters the page, as there are no safe way to store the data over time. This will be done by using JavaScript-functions that will track the amount of times "Buy tickets" are clicked for each category, and then update the variable.

Dynamic content / Ticket-carousel

The tickets will be shown for a given time period before changing to the next type. This will be triggered on load, and if someone press one of the ticket-types in the "ticket menu" then it will be turned off. This is due to usability, as you dont want it to change if for instance youre reading the ticket-information. This will be done using Javascript by combining a timer-function with change of the display attribute of each ticket type. More advanced animations may be implemented if we have time.
